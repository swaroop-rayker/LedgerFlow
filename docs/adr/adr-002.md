# ADR-002: Offline-First Architecture

## Context & Problem Statement
Personal financial data contains sensitive identifiers (bank accounts, transaction amounts, merchants). LedgerFlow must guarantee user privacy, data ownership, and instant offline accessibility.

## Proposed Options
1.  **Cloud-Only API Model**: Data resides on a backend server database.
2.  **Cloud-First Syncing Model**: Data is cached locally but synced to a central server.
3.  **Offline-First Local Model**: Data resides locally on the device with zero cloud connectivity by default.

## Decision Outcome
Selected Option: **Offline-First Local Model**

### Justification
*   **Absolute Privacy**: Excluding internet permission (`android.permission.INTERNET`) makes it physically impossible for the app to leak data or send telemetry.
*   **Zero Server Costs**: A self-contained application removes hosting and API maintenance overhead.
*   **Latency-Free**: Query execution matches device SQLite speeds without network delays.

## Consequences
*   **Pros**: Complete data ownership, immune to server security breaches, works everywhere, zero operational costs.
*   **Cons**: Data recovery relies entirely on manual encrypted backup exports; no cross-device sync without future local plugin integrations.
