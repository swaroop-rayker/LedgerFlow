# ADR-001: Kotlin + Jetpack Compose UI

## Context & Problem Statement
LedgerFlow requires a modern, responsive, and maintainable user interface system. The UI must support both light/dark modes, Material 3 design directives, and custom analytics graphs (via Vico charts) with low rendering latency and modular reusability.

## Proposed Options
1.  **Legacy XML Views + ViewBinding**: Traditional imperative layout system.
2.  **Jetpack Compose**: Modern declarative UI framework.
3.  **Cross-Platform Frameworks (e.g., Flutter, React Native)**: Frameworks to compile to iOS and Android.

## Decision Outcome
Selected Option: **Jetpack Compose**

### Justification
*   **Declarative Paradigm**: Greatly reduces boilerplate code. State transitions directly compute UI rendering, preventing synchronization bugs.
*   **Material 3 Out of the Box**: Native support for dynamic colors, elevations, and standard widgets.
*   **Direct Canvas Rendering**: Simplifies custom chart implementations and micro-animations.
*   **Future Readiness**: Easily scales to Compose Multiplatform if a desktop client is introduced later.

## Consequences
*   **Pros**: State-driven layout rendering, minimal XML files, rapid styling configurations, cleaner Kotlin MVVM architecture.
*   **Cons**: Requires learning compiler-driven recomposition optimizations to prevent unnecessary recompositions on heavy datasets.
